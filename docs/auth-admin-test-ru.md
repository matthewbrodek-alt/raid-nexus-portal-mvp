# Тестирование входа, личных кабинетов и админ-ролей

## Что уже подготовлено

- `/register` - регистрация через Firebase Auth email/password.
- `/login` - вход через Firebase Auth.
- `/dashboard` - личный кабинет обычного пользователя, доступен только авторизованным.
- `/admin` - админ-панель, доступна только ролям `admin` и `owner`.
- `users/{uid}` - профиль пользователя с ролью.
- `adminInvites/{email}` - приглашение администратора.
- `encryptedGameAccounts/{uid}` - зашифрованные игровые данные из регистрации.

## Роли

```text
user   - обычный пользователь.
admin  - администратор портала.
owner  - первый главный админ, может добавлять новых админов.
```

## Как назначить первого админа

1. Зарегистрируйся на сайте через `/register` с email первого владельца.
2. Открой Firebase Console -> Firestore Database -> `users`.
3. Найди документ пользователя по `uid`.
4. Измени поле `role` на `owner`.
5. Убедись, что поле `status` равно `active`.
6. Перезайди на сайт этим пользователем.
7. После входа открой `/admin`: должна открыться админ-панель.

## Как добавить второго админа

1. Войди как `owner`.
2. Открой `/admin`.
3. В блоке `Администраторы портала` введи email нового админа.
4. Нажми `Добавить админа`.
5. Новый пользователь должен войти или зарегистрироваться с этим email.
6. После входа его роль автоматически станет `admin`.

## Важное ограничение MVP

В MVP роли работают через Firestore-документы. Первый `owner` назначается вручную в Firebase Console, а клиентский bootstrap email не используется, чтобы не раскрывать роль владельца в браузерном коде. Для коммерческого production лучше перенести роли в Firebase Custom Claims через Cloud Functions или отдельный backend.

Для закрытого теста, Vercel preview и проверки сценариев текущей схемы достаточно.

## Проверка

1. Зарегистрируй обычного пользователя не из bootstrap email.
2. Убедись, что `/dashboard` открывается.
3. Убедись, что `/admin` показывает отказ доступа.
4. В Firebase Console вручную поставь первому администратору `role: "owner"`.
5. Перезайди owner-пользователем и убедись, что `/admin` открывается.
6. Добавь нового админа через `/admin`.
7. Войди новым email и проверь, что `/admin` открывается.
