# Эксплуатация и домен

## Перед публикацией

1. Проверь, что в проекте нет `.env`, `.env.local`, `.next`, `.vercel`, `node_modules` и архивов.
2. Заполни переменные окружения в Vercel.
3. Проверь Firebase rules из `firestore.rules`.
4. Проверь Cloudinary ключи для загрузки изображений и медиа.
5. Запусти локально `npm run build`.

## Основной поток заявок

Заявки создаются внутри Firestore и отображаются в админ-панели сайта.

`TOPUP_WEBHOOK_URL` нужен только если требуется дополнительно отправлять заявку во внешний сервис. Если переменная не задана, сайт продолжает работать без ошибки.

## Медиа

В репозитории уже лежат необходимые публичные ассеты из `public/`.

Большие файлы сейчас находятся в `public/videos/raffle/`. Если потребуется ускорить первую загрузку, лучше сжать эти ролики или оставить только реально используемые варианты.

## Проверка после деплоя

- Регистрация и вход.
- Личный кабинет.
- Создание заявки.
- Страница заявки и чат с менеджером.
- Админ-панель заявок.
- Загрузка изображений через Cloudinary.
- Новости, герои, чат, розыгрыш.

## Что делать при ошибках

- `auth/invalid-api-key`: неверный Firebase API key в Vercel.
- `auth/unauthorized-domain`: домен не добавлен в Firebase Authorized domains.
- Ошибка Cloudinary upload: проверь `NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME`, `CLOUDINARY_API_KEY`, `CLOUDINARY_API_SECRET`.
- Заявка не уходит во внешний сервис: проверь `TOPUP_WEBHOOK_URL`. Если внешний сервис не нужен, переменную можно оставить пустой.
